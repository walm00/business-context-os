# Onboarding Checklist

Progress tracker for CLEAR Context OS setup. Checked at session start.
When all items are complete, this file and its CLAUDE.md reference are deleted.

- [ ] Initial data points created and approved
- [ ] table-of-context.md created
- [ ] current-state.md created
- [ ] document-index.md generated (run: python .claude/scripts/build_document_index.py)

## Maintenance Setup
- [ ] Permissions verified — confirm `.claude/settings.json` has the dispatcher allowlist (run `python .claude/scripts/update.py` if anything's missing). Catalog: `docs/_bcos-framework/architecture/permissions-catalog.md`.
- [ ] Dispatcher task created — single `bcos-{project}` task that runs daily and coordinates all maintenance jobs. Use `context-onboarding` Step 6 (or read `docs/_bcos-framework/guides/scheduling.md`). Pick "Always allow" on the `mcp__scheduled-tasks__create_scheduled_task` prompt so future runs don't stall.
- [ ] First dispatcher run verified — ask Claude to "run today's maintenance now" and confirm a digest appears at `docs/_inbox/daily-digest.md` with no permission prompts firing.
